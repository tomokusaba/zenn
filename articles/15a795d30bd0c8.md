## はじめに

クラウドの導入を検討している組織にとって、「どこから始めればいいのか」「何を考慮すべきか」は大きな課題です。Microsoftは、Azure導入を成功に導くための包括的なガイドとして **Cloud Adoption Framework for Azure（CAF）** を提供しています。

この記事では、CAFの概要と、特に重要な「導入戦略（Strategy）」フェーズについて解説します。

:::message
本記事はLT（Lightning Talk）向けの内容をベースにしており、CAFの全体像のうち導入戦略の部分に焦点を当てています。
:::

## Cloud Adoption Framework（CAF）とは？

### 定義

**Cloud Adoption Framework for Azure** は、Azure導入を成功に導くための**Step by Stepガイド**です。

主な特徴：
- 🧑‍💼 **人（People）**：組織とスキルの準備
- 🔄 **プロセス（Process）**：ガバナンスと運用の確立
- 💻 **テクノロジー（Technology）**：技術的な実装と設計

これら3つの側面について、意思決定を支援するベストプラクティスが体系的にまとめられています。

### 対象者

CAFは **Azureを使用するすべてのユーザー** に提供されるガイドです。

| 組織タイプ | CAFの活用方法 |
|---|---|
| **スタートアップ** | クラウドネイティブなアプリケーションを最初から構築するためのアーキテクチャパターン |
| **エンタープライズ** | 複雑なマルチワークロード移行、ハイブリッドクラウドアーキテクチャ、大規模運用のガバナンス |
| **Azure初心者** | 構造化されたガイダンスを通じてAzure学習を加速 |
| **既存Azureユーザー** | 現在のデプロイメントの最適化、高度なコスト管理戦略の実装 |

## CAFの7つの手法（Methodology）

CAFは7つのコアメソドロジーで構成されています。最初の4つ（Strategy→Plan→Ready→Adopt）は順次実行され、残り3つ（Govern、Secure、Manage）はクラウドライフサイクル全体を通じて並行して運用されます。

:::message
※ Microsoft Learnのトレーニングモジュールでは「Teams and Roles（チームと役割）」を含め8つのメソドロジーとして言及されることもあります。
:::

### 基盤フェーズ（順次実行）

> **Strategy** → **Plan** → **Ready** → **Adopt**

これらは順番に進めていきます。戦略を立て、計画し、環境を準備し、実際に導入を進めます。

### 運用フェーズ（並行実行）

| Govern（ガバナンス） | Secure（セキュリティ） | Manage（運用） |
|:---:|:---:|:---:|
| リスク管理 | 環境保護 | 最適化 |

これら3つは基盤フェーズと並行して、クラウドライフサイクル全体を通じて継続的に実行されます。

### 各手法の概要

| CAF手法 | 説明 |
|---|---|
| 🚀 **Strategy（導入戦略）** | ビジネス上の正当な理由と期待される結果を定義します |
| 📝 **Plan（計画）** | 戦略で定義されたビジネス目標をもとにクラウド導入計画をします |
| 🛠️ **Ready（準備）** | Azure環境とランディングゾーンを定義します |
| 🌐 **Adopt（実施）** | Azureでマイグレーション、モダナイゼーション、イノベーションを進めます |
| 🛡️ **Govern（ガバナンス）** | リスクを管理するガバナンスを確立します |
| 🔒 **Secure（セキュリティ）** | Azure環境を保護します |
| ⚙️ **Manage（運用）** | クラウド環境を運用して最適化します |

## 導入戦略（Strategy）を深掘り

CAFの最初のステップである**導入戦略（Strategy）** は、クラウド導入の「なぜ」を定義する最も重要なフェーズです。

:::message alert
**大事なこと：目的を明確にする**

クラウド導入で最も重要なのは「**何の目的でクラウド導入するのかを明確にする**」ことです。

- 🏢 データセンターを廃止したいのか？
- ⚡ システムのデリバリーを高速化したいのか？
- 🤖 AIなどの新技術を提供して新しいビジネスを展開したいのか？

**なんらかのビジネス課題があって、それをどうにかするためにクラウドを導入する！**

裏を返せば、課題がなければわざわざお金をかけてクラウド導入する意味がありません。
:::

### 導入戦略の6つのステップ

導入戦略フェーズでは、以下の6つの手順を実行します。

| 手順 | タスク概要 |
|---|---|
| **1. 導入戦略を評価する** | 現在のクラウドへの成熟度を評価して改善のためのアクションを実行。[クラウド導入戦略エバリュエーター評価ツール](https://learn.microsoft.com/ja-jp/assessments/8fefc6d5-97ac-42b3-8e97-d82701e55bab/)を使用する |
| **2. 動機を決定する** | ビジネス目標との整合性を確保して投資収益率を最大化する |
| **3. ミッションと目標を定義** | 方向性と目的を定義して実行する手順の概要を定義する |
| **4. 戦略チームを定義** | クラウド戦略チームの動機とビジネス目標を定義する |
| **5. 組織を準備** | リーダーシップの賛同を得て、組織戦略との整合性を確保する |
| **6. 戦略に情報を提供する** | セキュリティ、AI導入、コスト効率、レジリエンス、サステナビリティに関する考慮事項を統合する |

それぞれのステップについて詳しく見ていきましょう。

### Step 1: 導入戦略を評価する

まず最初に、**現在のクラウド成熟度**を評価します。

Microsoftは「**クラウド導入戦略エバリュエーター**」という評価ツールを無料で提供しています。このツールを使うことで、組織の現状を客観的に把握し、改善すべきポイントを特定できます。

:::details クラウド導入戦略エバリュエーターとは？
オンラインで利用できる無料の評価ツールで、以下のような観点から組織のクラウド準備状況を評価します：
- 戦略的整合性
- 組織の準備状況
- 技術的準備状況
- 運用成熟度

👉 [クラウド導入戦略エバリュエーター](https://learn.microsoft.com/ja-jp/assessments/8fefc6d5-97ac-42b3-8e97-d82701e55bab/)
:::

### Step 2: 動機を決定する

**なぜクラウドを採用するのか？** この問いに明確に答えられることが重要です。

動機を明確にすることで、ビジネス目標との整合性を確保し、投資収益率（ROI）を最大化できます。

#### 動機の分類

クラウド導入の動機は大きく2つに分類できます：

**🔄 移行（Migration）志向の動機:**

| 動機 | 具体例 |
|---|---|
| 💰 **コスト削減** | ハードウェア更新コストの削減、運用コストの最適化 |
| 🏢 **データセンターの統合・廃止** | 老朽化したオンプレミス環境からの脱却 |
| ⏱️ **運用の最適化** | 運用負荷の軽減、自動化の推進 |
| 🔄 **アジリティの向上** | 迅速なリソースプロビジョニング |

**🚀 イノベーション志向の動機:**

| 動機 | 具体例 |
|---|---|
| 🎯 **新しいビジネス機能の実現** | クラウドネイティブな新サービスの開発 |
| 🌍 **市場投入までの時間短縮** | Time to Marketの改善 |
| 📊 **データ活用とAI導入** | Azure AI、Machine Learning の活用 |
| 🌱 **サステナビリティ** | 環境目標の達成、カーボンフットプリントの削減 |

### Step 3: ミッションと目標を定義

動機が明確になったら、次は**具体的なミッションと目標**を定義します。

**良い目標の条件（SMART）:**
- **S**pecific（具体的）
- **M**easurable（測定可能）
- **A**chievable（達成可能）
- **R**elevant（関連性がある）
- **T**ime-bound（期限がある）

:::message
**例：ミッションと目標の設定**

📌 **ミッション**: 「2年以内にオンプレミスのデータセンターを廃止し、すべてのワークロードをAzureに移行する」

🎯 **目標**:
- 6ヶ月以内に非本番環境の移行を完了
- 1年以内に本番環境の80%を移行
- 移行後のインフラコストを30%削減
:::

### Step 4: 戦略チームを定義

クラウド導入を成功させるには、**適切なチーム構成**が不可欠です。

#### 推奨されるチーム構成

| 役割 | 責任 |
|---|---|
| **エグゼクティブスポンサー** | 予算承認、組織全体の支援、意思決定 |
| **クラウドアーキテクト** | 技術的な方向性の決定、アーキテクチャ設計 |
| **プロジェクトマネージャー** | 計画の策定と実行管理 |
| **セキュリティリード** | セキュリティ要件の定義と実装 |
| **運用チーム** | 日々の運用と監視 |
| **ビジネスオーナー** | ビジネス要件の定義、ROIの評価 |

### Step 5: 組織を準備

技術だけでなく、**組織としての準備**も重要です。

#### リーダーシップの賛同を得る

リーダーシップの賛同がないと、以下のリスクが発生します：
- 必要なリソース（予算、人材、時間）が確保できない
- 従業員の変革への抵抗が高まる
- プロジェクトが途中で頭打ちになる

#### 組織戦略との整合

クラウド導入戦略は、以下の既存戦略と整合させる必要があります：

- ✅ **ビジネス戦略**: 組織の高レベルなミッションと目標
- ✅ **デジタル戦略**: テクノロジーによるビジネス変革
- ✅ **IT戦略**: インフラとリソースの管理
- ✅ **運用モデル**: クラウド運用の準備状況

### Step 6: 戦略に情報を提供する（Inform your strategy）

最後に、**重要な考慮事項を戦略に統合**します。

このステップでは、以下の分野について戦略的な考慮事項を収集し、戦略に反映させます：

| 考慮事項 | 内容 |
|---|---|
| 💵 **コスト効率** | FinOpsの実践、クラウド投資の最適化 |
| 🤖 **AI** | Azure AI サービスの活用計画、機械学習の導入 |
| 🛡️ **レジリエンス** | 障害耐性と事業継続性の確保 |
| 🔒 **セキュリティ** | Zero Trustアプローチの採用 |
| 🌿 **サステナビリティ** | 環境への配慮、カーボンフットプリントの削減 |

## 次のステップ：計画（Plan）へ

導入戦略フェーズで**クラウド導入の動機が明らかになりました**。

次の「計画（Plan）」フェーズでは、この戦略を実現するための**具体的なソリューションと実行計画**を策定していきます。

:::message
📝 **Plan フェーズでやること**
- ワークロードの棚卸しと優先順位付け
- 移行戦略の選択（リホスト、リプラットフォーム、リファクタリングなど）
- タイムラインとマイルストーンの設定
- 必要なリソースと予算の見積もり

👉 **続きはこちら**: [Cloud Adoption Framework（計画編）](https://zenn.dev/tomokusaba/articles/b8590045af597c)
:::

## CAFで提供されるシナリオ

CAFは一般的なガイダンスに加えて、特定のクラウドテクノロジーを導入するための**シナリオベースのガイダンス**も提供しています。

| シナリオ | 説明 |
|---|---|
| **AzureでのAI導入** | AzureでAIを正しい方法で使用する方法 |
| **Azure VMware Solution** | VMwareワークロードをAzureに移行する |
| **ハイブリッドとマルチクラウド** | オンプレミスとクラウドのシステムを接続する |
| **Azure上のOracle DB** | AzureでOracleワークロードを移行および管理する |
| **SAP on Azure** | クラウドネイティブツールを移行してAzureでSAPを実行します |
| **Virtual Desktop** | Azure Virtual Desktopに移行して大規模なデスクトップを管理する |
| **HPC（ハイパフォーマンスコンピューティング）** | Azureでハイパフォーマンスコンピューティングワークロードを実行します |

## Well-Architected Frameworkとの関係

CAFと一緒によく語られる **Well-Architected Framework（WAF）** についても触れておきましょう。

| 観点 | CAF (Cloud Adoption Framework) | WAF (Well-Architected Framework) |
|---|---|---|
| **焦点** | どうやってクラウドを導入するか | どうやって良いアーキテクチャを設計するか |
| **スコープ** | 組織・プロセス・技術の変革をガイド | 5つの柱（信頼性、セキュリティ、コスト最適化、オペレーショナルエクセレンス、パフォーマンス効率） |
| **カバー範囲** | 導入のライフサイクル全体 | 個々のワークロードの設計と評価 |

:::message
**使い分けのポイント**
- **CAF** でクラウド導入を計画し
- **WAF** で各ワークロードを設計する

両者は補完関係にあり、組み合わせて使うことで効果的なクラウド導入が実現できます。
:::

## セキュリティ戦略：Zero Trust

CAFのセキュリティ考慮事項では、**Zero Trust** の原則が重要視されています。

### Zero Trustの3つの原則

1. **明示的に検証する（Verify explicitly）**
   - すべてのデータポイントに基づいて常に認証と認可を行う

2. **最小権限アクセス（Use least privilege）**
   - Just-In-Time、Just-Enough-Access、リスクベースの適応ポリシーでユーザーアクセスを制限

3. **侵害を想定する（Assume breach）**
   - 影響範囲を最小化し、アクセスをセグメント化
   - エンドツーエンドの暗号化を検証し、分析を使用して可視性を獲得

## まとめ

Cloud Adoption Framework（CAF）は、Azureクラウド導入を成功に導くための包括的なガイドです。

**今日のポイント:**

- ☁️ CAFは「人・プロセス・テクノロジー」の3側面から導入を支援
- 🎯 導入戦略（Strategy）は「なぜクラウドを採用するのか」を明確にする最初のステップ
- 🔄 7つの手法を通じて、計画から運用までをカバー
- 🏗️ WAFと組み合わせることで、導入と設計の両方をガイド
- 🔒 Zero Trustの原則を取り入れたセキュリティ戦略
